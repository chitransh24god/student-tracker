STARTUP FILES (Run These):
════════════════════════════════════════════════════════════════════════════════

  🪟 START.bat ⭐
     Platform: Windows only
     Usage: Double-click
     Features: Progress indicators [1/5], colored output, error handling
     Best for: First-time users on Windows
     Time to completion: 2-3 minutes

  🪟 run.bat
     Platform: Windows only
     Usage: Double-click or cmd.exe
     Features: Same as START.bat, alternative option
     Best for: Windows users who prefer cmd.exe

  🍎 run.sh
     Platform: Mac/Linux
     Usage: bash run.sh (from terminal)
     Features: Cross-platform compatibility, error handling
     Best for: Mac and Linux users
     First time: chmod +x run.sh (make executable)

  🐍 run.py
     Platform: All platforms (requires Python)
     Usage: python run.py (from terminal)
     Features: Universal launcher, 500+ lines, robust
     Best for: Python developers, or if other scripts fail

❓ FAQ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: Which startup file should I use?
A: START.bat on Windows, bash run.sh on Mac/Linux

Q: It's not working, what should I do?
A: Read STARTUP_GUIDE.txt "If Something Goes Wrong" section

Q: How do I change the database?
A: Delete attendance.db and restart - it will recreate with sample data

Q: How do I reset everything?
A: Delete venv/ and attendance.db, then restart

Q: Can I use this in production?
A: Not recommended - SQLite is single-user. Use PostgreSQL for production.

Q: How do I backup my data?
A: Copy attendance.db to another location

Q: Can I access from another computer?
A: No - localhost:5000 only works on your computer. Modify app.py for LAN access.

Q: What's the default admin password?
A: There's no admin account - create one by signing up with role=Faculty

Q: Can I delete the venv folder?
A: Yes - it will be recreated on next startup. Not needed for backup.

Q: How do I update the app?
A: Replace the files (keep attendance.db and venv/ to preserve data)

Q: Is my data secure?
A: Passwords are hashed. Database is local. No transmission = no interception.
