CSV FILE REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✓ Must be CSV format (.csv file)
✓ Must have 3 columns in this order: name, email, password
✓ First row must be header: name,email,password
✓ Name: up to 150 characters
✓ Email: must be valid and unique (no duplicates)
✓ Password: minimum 6 characters

Example of GOOD CSV:
  name,email,password
  John Doe,john@school.com,securepass123
  Jane Smith,jane@school.com,anotherpass99
  Bob Johnson,bob@school.com,bobpass777

Example of BAD CSV:
  name,email,password
  John Doe,john@school.com
  Jane Smith
  ,invalid@,weakpass

Why would it fail?
  ✗ Row 2: Missing password (incomplete)
  ✗ Row 3: Missing all data (incomplete)
  ✗ Row 4: Invalid email format (invalid)

Result: "Added 1 student (Skipped: 3)"
        Only the first row would be added

TIPS & TRICKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 Tip 1: Prepare CSV in Excel
   • Open Excel
   • Create 3 columns: name, email, password
   • Add data rows
   • File → Save As → Format: CSV (Comma delimited)
   • Upload to Bulk Upload page

💡 Tip 2: Create systematic email addresses
   Good:  student001@school.com, student002@school.com
   Better: john.smith@school.com, jane.doe@school.com

💡 Tip 3: Use meaningful names
   Include first and last names for easier searching
   Easier to find: "John Smith" instead of "js1"

💡 Tip 4: Backup before bulk actions
   Before uploading 100+ students:
   • Download the database file (attendance.db)
   • Keep a backup copy
   • Helps if something goes wrong

💡 Tip 5: Test with one student first
   Don't upload 1000 students on first try
   Add 1 student via form first
   Then upload 10 via CSV
   Then do bulk uploads
