SECURITY FEATURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ Authentication
   • Session-based login system
   • Passwords hashed with Werkzeug
   • @login_required decorator on protected routes
   • Session cookies with HTTPOnly & SameSite flags

✅ Authorization  
   • @faculty_required decorator on all admin routes
   • Only faculty can access admin panel
   • Students cannot see admin menu

✅ Data Validation
   • Client-side: HTML5 validation
   • Server-side: All inputs validated
   • Email: Format check, uniqueness at DB level
   • Password: Minimum 6 characters

✅ Database Protection
   • Email unique constraint in User table
   • Foreign key constraints for data integrity
   • Cascade deletion when students deleted
   • All queries use SQLAlchemy ORM (SQL injection safe)

✅ File Upload Protection
   • Only .csv files accepted
   • No executable file uploads
   • Size limits enforced
   • Content validation before import
